# <a name="calling-sqlgetdiagfield"></a>Chamar SQLGetDiagField
Quando um aplicativo ODBC *3.x* chama **SQLGetDiagField** em um driver ODBC *2.x,* o driver retornará SQL_SUCCESS e as informações apropriadas no * \*DiagInfoPtr* se o argumento *DiagIdentifier* for SQL_DIAG_CLASS_ORIGIN, SQL_DIAG_CLASS_SUBCLASS_ORIGIN, SQL_DIAG_CONNECTION_NAME, SQL_DIAG_MESSAGE_TEXT, SQL_DIAG_NATIVE, SQL_DIAG_NUMBER, SQL_DIAG_RETURNCODE, SQL_DIAG_SERVER_NAME ou SQL_DIAG_SQLSTATE. Todos os outros campos de diagnóstico retornarão SQL_ERROR.
