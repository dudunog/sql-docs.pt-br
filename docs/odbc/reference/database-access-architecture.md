# <a name="database-access-architecture"></a>Arquitetura de acesso ao banco de dados
Uma das questões no desenvolvimento da ODBC foi qual parte da arquitetura de acesso ao banco de dados para padronizar. As interfaces de programação SQL descritas na seção anterior - SQL embarcada, módulos SQL e CLIs - são apenas uma parte desta arquitetura. Na verdade, como o ODBC tinha como objetivo principal conectar aplicativos pessoais baseados em computadores a Minicomputadores e DBMSs de mainframe, havia também uma série de componentes de rede, alguns dos quais poderiam ser padronizados.  
  
 Esta seção contém os seguintes tópicos.
